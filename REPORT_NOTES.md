# REPORT_NOTES: проект BankVS

## 1. Общее описание проекта

BankVS - учебная банковская система на C++17 с графическим интерфейсом WinAPI. Программа позволяет вести набор банковских счетов, выполнять основные операции с деньгами, сохранять данные в файл, загружать их при следующем запуске, смотреть историю операций, создавать выписки и просматривать простую статистику.

Основные возможности:

- создание трех типов счетов: накопительный, расчетный и кредитный;
- пополнение счета;
- снятие денег со счета;
- перевод между счетами;
- начисление процентов по одному счету или по всем счетам;
- поиск счета по ID, владельцу или типу;
- удаление счета;
- просмотр таблицы счетов;
- просмотр истории операций;
- сохранение и загрузка данных из `bank_data.txt`;
- автоматическая загрузка при запуске и автосохранение после изменяющих операций;
- создание выписки `statement_ID.txt` в папке `statements`;
- текстовая и графическая статистика.

Проект многофайловый, потому что разные части программы разделены по ответственности. Банковская логика вынесена в классы `Account`, `Bank`, `FileManager`, наследников счетов и `Transaction<T>`. Интерфейс вынесен в `MainWindow`, отдельные dialog-файлы и вспомогательные GUI-утилиты. Такое разделение проще объяснять, тестировать и дорабатывать: например, правила снятия денег меняются в классах счетов, а расположение кнопок - в `MainWindow.cpp` или соответствующем диалоге.

WinAPI используется для создания настоящего Windows-приложения без внешних GUI-фреймворков. В коде напрямую применяются `CreateWindowExW`, `CreateWindowW`, `WNDCLASSW`, обработка сообщений `WM_CREATE`, `WM_COMMAND`, `WM_SIZE`, `WM_PAINT`, `WM_CLOSE`, элементы управления `BUTTON`, `EDIT`, `STATIC`, `COMBOBOX`, `ListView`, а также GDI-функции для рисования графика.

## 2. Общая архитектура

Проект удобно разделить на две большие части.

### Банковская логика

К этой части относятся файлы:

- `Account.h`, `Account.cpp` - базовый абстрактный класс счета;
- `SavingsAccount.h`, `SavingsAccount.cpp` - накопительный счет;
- `CheckingAccount.h`, `CheckingAccount.cpp` - расчетный счет с овердрафтом;
- `CreditAccount.h`, `CreditAccount.cpp` - кредитный счет с кредитным лимитом и процентами по долгу;
- `Transaction.h` - шаблонный класс банковской операции;
- `Bank.h`, `Bank.cpp` - центральный класс банка, хранит счета и выполняет операции;
- `Exceptions.h` - пользовательские исключения;
- `FileManager.h`, `FileManager.cpp` - сохранение и загрузка банка;
- `StatisticsVisualizer.h`, `StatisticsVisualizer.cpp` - текстовая статистика и текстовые диаграммы.

Эта часть не зависит от конкретного окна. Она описывает правила: какие счета существуют, как менять баланс, когда бросать исключение, как сохранить данные, как загрузить данные, как сформировать историю и выписку.

### Графический интерфейс

К GUI относятся:

- `main.cpp` - точка входа Windows-приложения;
- `MainWindow.h`, `MainWindow.cpp` - главное окно, меню, кнопки, таблица счетов, статус и автосохранение;
- `DialogUtils.h`, `DialogUtils.cpp` - общие функции для окон: создание контролов, чтение ввода, конвертация UTF-8/wide string, вывод ошибок;
- `UiHelpers.h`, `UiHelpers.cpp` - форматирование денег, процентов, типов счетов, кратких описаний счета;
- `CreateAccountDialog.h`, `CreateAccountDialog.cpp` - окно создания счета;
- `MoneyOperationDialog.h`, `MoneyOperationDialog.cpp` - окно пополнения и снятия;
- `TransferDialog.h`, `TransferDialog.cpp` - окно перевода;
- `InterestDialog.h`, `InterestDialog.cpp` - окно начисления процентов;
- `HistoryDialog.h`, `HistoryDialog.cpp` - окно истории операций;
- `SearchDialog.h`, `SearchDialog.cpp` - окно поиска;
- `DeleteAccountDialog.h`, `DeleteAccountDialog.cpp` - окно удаления счета;
- `StatementDialog.h`, `StatementDialog.cpp` - окно создания выписки;
- `StatisticsDialog.h`, `StatisticsDialog.cpp` - окно статистики и GDI-график;
- `BankVS.h`, `Resource.h`, `framework.h`, `targetver.h` - вспомогательные файлы Visual Studio/WinAPI.

GUI получает ввод от пользователя, вызывает методы `Bank`, показывает результат, обновляет таблицу и статистику. Важная идея: кнопки и окна сами не должны хранить банковские правила. Они читают данные и передают их в `Bank`.

## 3. Структура файлов проекта

### `main.cpp`

Отвечает за запуск приложения. Здесь объявлена функция `wWinMain`, создается объект `Bank`, создается `MainWindow`, вызываются `Create` и `Run`.

Основные элементы:

- `int WINAPI wWinMain(...)`;
- локальный объект `Bank bank`;
- локальный объект `MainWindow mainWindow(hInstance, bank)`.

Связан с `Bank.h` и `MainWindow.h`.

При доработках здесь обычно меняют только стартовую инициализацию приложения: например, если нужно создать другой главный объект, передать параметры запуска или изменить поведение при ошибке создания окна.

### `Account.h`

Описывает абстрактный базовый класс `Account`. Это общий интерфейс для всех типов счетов.

Основные поля:

- `id` - номер счета;
- `ownerName` - имя владельца;
- `balance` - баланс;
- `history` - `std::vector<Transaction<double>>` с историей операций.

Основные методы:

- `getId`, `getOwnerName`, `getBalance`, `getHistory`;
- `deposit`;
- чисто виртуальные `withdraw`, `applyInterest`, `getType`, `canWithdraw`, `serialize`;
- `addTransaction`;
- `print`;
- `currentDate`;
- `changeBalance`;
- `operator<<` для вывода счета.

Связан с `Transaction.h`, `Bank.h` и всеми наследниками счетов.

Что менять при доработках: общие поля счета, общую логику пополнения, формат печати счета, общий интерфейс для новых типов счетов. Если добавить обязательный метод для всех счетов, его объявляют здесь.

### `Account.cpp`

Реализует общую логику `Account`.

Основные функции:

- конструктор `Account::Account`;
- `Account::currentDate` - получает текущую дату в формате `YYYY-MM-DD`;
- `Account::changeBalance` - изменяет баланс;
- геттеры;
- `Account::deposit` - проверяет положительную сумму, увеличивает баланс, добавляет транзакцию `Deposit`;
- `Account::addTransaction`;
- `Account::print`;
- `operator<<`.

Связан с `Account.h` и `Exceptions.h`.

Что менять: правила общего пополнения, формат даты, формат базового вывода счета. Логику снятия здесь не меняют, потому что она разная у наследников.

### `SavingsAccount.h`

Описывает накопительный счет `SavingsAccount`, наследник `Account`.

Основное поле:

- `interestRate` - процентная ставка.

Основные методы:

- `canWithdraw`;
- `getInterestRate`;
- `getType`;
- `withdraw`;
- `applyInterest`;
- `print`;
- `serialize`.

Связан с `Account.h`.

Что менять: интерфейс накопительного счета, дополнительные поля накопительного счета, правила, которые должны быть доступны другим файлам.

### `SavingsAccount.cpp`

Реализует накопительный счет.

Главная логика:

- снятие разрешено только если `balance >= amount`;
- баланс не может уйти в минус;
- проценты начисляются только при положительном балансе: `interest = balance * interestRate`;
- при начислении добавляется транзакция `Interest`;
- `serialize` сохраняет тип `Savings`, ID, владельца, баланс, ставку и историю.

Связан с `SavingsAccount.h`, `Exceptions.h`, `Transaction.h`.

Что менять: правило снятия с накопительного счета, формулу процентов, формат сохранения накопительного счета.

### `CheckingAccount.h`

Описывает расчетный счет `CheckingAccount`, наследник `Account`.

Основное поле:

- `overdraftLimit` - разрешенный уход в минус.

Основные методы:

- `canWithdraw`;
- `getOverdraftLimit`;
- `getType`;
- `withdraw`;
- `applyInterest`;
- `print`;
- `serialize`.

Связан с `Account.h`.

Что менять: интерфейс расчетного счета и дополнительные параметры расчетного счета.

### `CheckingAccount.cpp`

Реализует расчетный счет.

Главная логика:

- снятие разрешено, если после операции баланс не меньше `-overdraftLimit`;
- при снятии добавляется транзакция `Withdraw`;
- `applyInterest` пустой, потому что у расчетного счета проценты не начисляются;
- `serialize` сохраняет тип `Checking`, ID, владельца, баланс, овердрафт и историю.

Связан с `CheckingAccount.h`, `Exceptions.h`, `Transaction.h`.

Что менять: правило овердрафта, текст ошибок по расчетному счету, формат сохранения расчетного счета.

### `CreditAccount.h`

Описывает кредитный счет `CreditAccount`, наследник `Account`.

Основные поля:

- `creditLimit` - максимальный кредитный лимит;
- `interestRate` - процентная ставка по долгу.

Основные методы:

- `canWithdraw`;
- `getCreditLimit`;
- `getInterestRate`;
- `getType`;
- `deposit`;
- `withdraw`;
- `applyInterest`;
- `print`;
- `serialize`.

Связан с `Account.h`.

Что менять: интерфейс кредитного счета, новые параметры кредитного счета.

### `CreditAccount.cpp`

Реализует кредитный счет.

Главная логика:

- снятие разрешено, если баланс после снятия не меньше `-creditLimit`;
- пополнение использует базовый `Account::deposit`;
- проценты начисляются только при отрицательном балансе;
- сумма процентов считается как `abs(balance) * interestRate`;
- проценты увеличивают долг, поэтому баланс уменьшается на сумму процентов;
- добавляется транзакция `Credit Interest`;
- `serialize` сохраняет тип `Credit`, ID, владельца, баланс, кредитный лимит, ставку и историю.

Связан с `CreditAccount.h`, `Exceptions.h`, `Transaction.h`.

Что менять: правила кредитного лимита, формулу процентов по долгу, формат сохранения кредитного счета.

### `Transaction.h`

Содержит шаблонный класс `Transaction<T>`.

Основные поля:

- `amount` - сумма операции;
- `date` - дата;
- `type` - тип операции: `Deposit`, `Withdraw`, `Transfer In`, `Transfer Out`, `Interest`, `Credit Interest`;
- `relatedAccountId` - связанный счет для переводов, либо `-1`.

Основные методы:

- конструкторы;
- `getAmount`, `getDate`, `getType`, `getRelatedAccountId`;
- `writeToFile`;
- `readFromFile`;
- `operator<<`;
- внутренние `encodeText` и `decodeText`, которые заменяют пробелы на `_` при сохранении.

Связан с `Account.h`, `Bank.cpp`, наследниками счетов.

Что менять: формат записи транзакции, поля транзакции, формат вывода операции.

### `Bank.h`

Описывает центральный класс `Bank`.

Основное поле:

- `std::map<int, std::shared_ptr<Account>> accounts` - счета по ID.

Основные методы:

- создание счетов: `createSavingsAccount`, `createCheckingAccount`, `createCreditAccount`;
- поиск: `getAccount`;
- удаление: `removeAccount`;
- операции: `deposit`, `withdraw`, `transfer`;
- проценты: `applyInterestToAccount`, `applyInterestToAllAccounts`;
- текстовый вывод: `getAllAccountsText`, `getAccountHistoryText`;
- выписка: `generateStatement`;
- доступ к контейнеру: `getAccounts`;
- очистка: `clear`;
- перегруженный `operator+` для перевода через `TransferRequest`;
- операторы `operator<<` и `operator>>` для сохранения и загрузки.

Связан со всеми классами банковской логики и с GUI.

Что менять: публичный API банка, добавление новых операций, новый способ доступа к счетам.

### `Bank.cpp`

Реализует основную банковскую логику.

Главная логика:

- `validateNewAccount` проверяет положительный ID и уникальность;
- методы создания счетов проверяют начальный баланс, лимиты и ставки;
- `getAccount` бросает исключение, если счет не найден;
- `deposit` вызывает виртуальный/обычный метод счета;
- `withdraw` вызывает виртуальный `withdraw`, поэтому работает полиморфизм;
- `transfer` вызывает перегруженный `operator+`;
- `operator+` проверяет сумму, разные ID, наличие счетов и возможность снятия;
- при переводе баланс меняется напрямую через `changeBalance`, чтобы не создавать обычные `Deposit`/`Withdraw`;
- для перевода создаются транзакции `Transfer Out` и `Transfer In`;
- `generateStatement` создает папку при необходимости и записывает выписку;
- `operator<<` сохраняет все счета;
- `operator>>` загружает счета и историю из файла.

Связан с `SavingsAccount.h`, `CheckingAccount.h`, `CreditAccount.h`, `Exceptions.h`, `Transaction.h`, `FileManager.cpp`, GUI-диалогами.

Что менять: правила банковских операций, формат списка счетов, формат истории, формат выписки, формат сериализации.

### `Exceptions.h`

Содержит пользовательские исключения:

- `InsufficientFundsException` - недостаточно средств или превышен лимит;
- `InvalidAccountException` - неверный счет, счет не найден, неверный ID;
- `InvalidAmountException` - неверная сумма или параметры.

Все наследуются от `std::runtime_error`.

Связан с банковской логикой и `DialogUtils.cpp`, где исключения переводятся в понятные сообщения для пользователя.

Что менять: добавлять новые типы ошибок или менять базовый тип исключений.

### `FileManager.cpp`

Реализует сохранение и загрузку через файловые потоки.

Главная логика:

- `saveBank` открывает `std::ofstream`, проверяет файл и пишет `file << bank`;

Связан с `Bank.cpp`, потому что реальные операторы сохранения/загрузки реализованы в `Bank.cpp`.

Что менять: обработку ошибок открытия файла, способ открытия, имя или кодировку файла, если путь будет передаваться иначе.

## 4. Основные классы

### `Account`

Назначение: общий базовый класс для всех счетов.

Основные поля:

- `id`;
- `ownerName`;
- `balance`;
- `history`.

Основные методы:

- `deposit`;
- `withdraw`;
- `applyInterest`;
- `getType`;
- `canWithdraw`;
- `serialize`;
- `addTransaction`;
- `print`.

Почему нужен: чтобы все типы счетов имели общий интерфейс. Благодаря этому `Bank` может хранить разные счета в одном контейнере как `std::shared_ptr<Account>` и вызывать виртуальные методы.

Где используется: в `Bank`, `StatisticsVisualizer`, `UiHelpers`, наследниках счетов и GUI-таблице.

### `SavingsAccount`

Назначение: накопительный счет с процентной ставкой.

Основное поле:

- `interestRate`.

Основные методы:

- `withdraw` - не разрешает уходить в минус;
- `applyInterest` - начисляет проценты на положительный баланс;
- `getType` возвращает `Savings`;
- `serialize` сохраняет счет.

Почему нужен: демонстрирует наследование и отдельное поведение счета с процентами.

Где используется: создается в `Bank::createSavingsAccount`, отображается в `MainWindow` и статистике.

### `CheckingAccount`

Назначение: расчетный счет с овердрафтом.

Основное поле:

- `overdraftLimit`.

Основные методы:

- `withdraw` - разрешает уйти в минус только до лимита;
- `applyInterest` - пустая реализация;
- `getType` возвращает `Checking`;
- `serialize`.

Почему нужен: показывает, что разные наследники могут по-разному реализовывать один и тот же виртуальный метод.

Где используется: создается в `Bank::createCheckingAccount`, отображается в таблице и поиске.

### `CreditAccount`

Назначение: кредитный счет с лимитом и процентами по долгу.

Основные поля:

- `creditLimit`;
- `interestRate`.

Основные методы:

- `withdraw` - проверяет кредитный лимит;
- `deposit` - использует базовое пополнение;
- `applyInterest` - если баланс отрицательный, увеличивает долг;
- `getType` возвращает `Credit`;
- `serialize`.

Почему нужен: демонстрирует более сложную модель счета, где проценты начисляются не на положительный баланс, а на задолженность.

Где используется: создается в `Bank::createCreditAccount`, учитывается в `UiHelpers` и статистике.

### `Bank`

Назначение: центральный класс управления счетами.

Основное поле:

- `accounts`, карта `ID -> shared_ptr<Account>`.

Основные методы:

- создание счетов;
- получение счета;
- удаление;
- пополнение;
- снятие;
- перевод;
- начисление процентов;
- генерация выписки;
- сохранение/загрузка через операторы.

Почему нужен: концентрирует банковские операции в одном месте. GUI не должен напрямую менять баланс счетов, он обращается к `Bank`.

Где используется: создается в `main.cpp`, передается в `MainWindow`, а дальше во все диалоги.

Назначение: описывает одну операцию по счету.

Основные поля:

- `amount`;

Основные методы:

## 5. Как работают основные сценарии

### Создание счета

1. Пользователь нажимает кнопку "Создать счёт" в главном окне или выбирает соответствующий пункт меню.
2. `MainWindow::ProcessCommand` получает команду `ID_BTN_CREATE_ACCOUNT`.
3. Вызывается `CreateAccountDialog::Show`.
4. В диалоге пользователь выбирает тип счета и вводит ID, владельца, начальный баланс и дополнительные поля.
5. `CreateAccountDialog.cpp` читает ввод через `DialogUtils::ReadInt`, `ReadDouble`, `ParsePercentInput`, `ReadRequiredUtf8Text`.
6. В зависимости от выбранного типа вызывается:
   - `Bank::createSavingsAccount`;
   - `Bank::createCheckingAccount`;
   - `Bank::createCreditAccount`.
7. `Bank` проверяет ID, баланс, лимиты и ставку.
8. Создается объект `SavingsAccount`, `CheckingAccount` или `CreditAccount` через `std::make_shared`.
9. Счет помещается в `std::map accounts`.
10. Диалог вызывает `outputCallback`.
11. В `MainWindow` этот callback обновляет таблицу, статистику, выбранный счет и статус.
12. Так как операция изменяет данные, вызывается `AutoSave`, и банк сохраняется в `bank_data.txt`.

### Пополнение

1. Пользователь нажимает "Пополнить счёт" или кнопку пополнения выбранного счета.
2. `MainWindow::ProcessCommand` открывает `MoneyOperationDialog` с типом `MoneyOperationType::Deposit`.
3. Диалог читает ID и сумму.
4. Вызывается `Bank::deposit(id, amount)`.
5. `Bank` получает счет через `getAccount(id)`.
6. У счета вызывается `deposit`.
7. Базовая реализация `Account::deposit` проверяет, что сумма положительная, увеличивает баланс и добавляет транзакцию `Deposit`.
8. Callback обновляет интерфейс.
9. После успешной операции `AutoSave` сохраняет данные.

### Снятие

1. Пользователь нажимает "Снять деньги" или кнопку снятия выбранного счета.
2. Открывается `MoneyOperationDialog` с типом `MoneyOperationType::Withdraw`.
3. Диалог читает ID и сумму.
4. Вызывается `Bank::withdraw(id, amount)`.
5. `Bank` получает счет и вызывает виртуальный метод `withdraw`.
6. Если счет накопительный, работает `SavingsAccount::withdraw`: баланс не может стать отрицательным.
7. Если счет расчетный, работает `CheckingAccount::withdraw`: учитывается `overdraftLimit`.
8. Если счет кредитный, работает `CreditAccount::withdraw`: учитывается `creditLimit`.
9. При успехе баланс уменьшается и добавляется транзакция `Withdraw`.
10. Callback обновляет таблицу и статистику.
11. `AutoSave` сохраняет данные.

### Перевод

1. Пользователь нажимает "Перевести деньги".
2. `MainWindow::ProcessCommand` открывает `TransferDialog`.
3. Диалог читает ID отправителя, ID получателя и сумму.
4. Вызывается `Bank::transfer(fromId, toId, amount)`.
5. `Bank::transfer` вызывает перегруженный оператор: `*this + TransferRequest{fromId, toId, amount}`.
6. `Bank::operator+` проверяет:
   - сумма положительная;
   - отправитель и получатель разные;
   - оба счета существуют;
   - со счета отправителя можно снять сумму через `canWithdraw`.
7. Баланс отправителя уменьшается через `changeBalance(-amount)`.
8. Баланс получателя увеличивается через `changeBalance(amount)`.
9. Для отправителя создается транзакция `Transfer Out` со связанным ID получателя.
10. Для получателя создается транзакция `Transfer In` со связанным ID отправителя.
11. Callback обновляет интерфейс.
12. `AutoSave` сохраняет данные.

`operator+` здесь нужен как демонстрация перегрузки операторов: перевод можно представить как добавление к банку объекта-запроса `TransferRequest`.

### Начисление процентов

1. Пользователь нажимает "Начислить проценты".
2. Открывается `InterestDialog`.
3. Можно начислить проценты по одному счету или по всем счетам.
4. Для одного счета вызывается `Bank::applyInterestToAccount(id)`.
5. Для всех счетов вызывается `Bank::applyInterestToAllAccounts()`.
6. В обоих случаях срабатывает виртуальный метод `applyInterest`.

Для `SavingsAccount`:

- проценты начисляются только если баланс положительный;
- сумма процентов: `balance * interestRate`;
- баланс увеличивается;
- добавляется транзакция `Interest`.

Для `CheckingAccount`:

- метод `applyInterest` реализован, но ничего не делает;
- это нужно, потому что метод виртуальный и обязателен для всех наследников.

Для `CreditAccount`:

- проценты начисляются только если баланс отрицательный;
- сумма процентов: `abs(balance) * interestRate`;
- баланс уменьшается, то есть долг растет;
- добавляется транзакция `Credit Interest`.

После успешного начисления интерфейс обновляется и выполняется `AutoSave`.

### Сохранение и загрузка

Автозагрузка:

1. В `MainWindow::CreateControls` после создания интерфейса вызывается `AutoLoad`.
2. `AutoLoad` проверяет существование `bank_data.txt`.
3. Если файл есть, вызывается `FileManager::loadBank(bank, DATA_FILE)`.
4. `FileManager::loadBank` открывает `std::ifstream`.
5. Затем выполняется `file >> bank`.
6. `operator>>` в `Bank.cpp` читает количество счетов, тип каждого счета, поля счета и список транзакций.
7. Для восстановления объектов вызываются методы создания счетов.
8. Транзакции читаются через `Transaction<double>::readFromFile`.

Автосохранение:

1. В `MainWindow::ProcessCommand` для изменяющих операций используется `mutatingOutputCallback`.
2. После успешной операции callback обновляет интерфейс.
3. Если сообщение не является ошибкой, вызывается `AutoSave`.
4. `AutoSave` вызывает `FileManager::saveBank(bank, DATA_FILE)`.
5. `FileManager::saveBank` открывает `std::ofstream`.
6. Затем выполняется `file << bank`.
7. `operator<<` в `Bank.cpp` записывает все счета.
8. Для каждого счета вызывается виртуальный `serialize`, поэтому каждый тип записывает свои дополнительные поля.
9. Транзакции сохраняются через `Transaction<double>::writeToFile`.

Ручное сохранение и загрузка также находятся в `MainWindow::ProcessCommand`: команды `ID_BTN_SAVE` и `ID_BTN_LOAD`.

### Выписка

1. Пользователь нажимает "Выписка" или кнопку выписки выбранного счета.
2. Открывается `StatementDialog` или вызывается `MainWindow::GenerateStatementForAccount`.
3. Формируется путь `statements/statement_ID.txt`.
4. Если файл уже есть, пользователь подтверждает перезапись.
5. Вызывается `Bank::generateStatement(id, filename)`.
6. В `Bank::generateStatement` проверяется, что счет существует.
7. Если у пути есть родительская папка, вызывается `std::filesystem::create_directories`.
8. Создается `std::ofstream`.
9. В файл записываются ID счета, владелец, тип, текущий баланс и история операций.
10. После создания программа предлагает открыть папку.
11. Папка `statements` открывается через `ShellExecuteW`.

Папка выписок находится в корне проекта/рабочей папке приложения: `statements`.

### Статистика

Главная таблица:

1. `MainWindow::RefreshAccountTable` очищает `ListView`.
2. Перебирает `bank.getAccounts()`.
3. Для каждого счета добавляет строку: ID, тип, владелец, баланс, ставка, лимит/овердрафт.
4. Для форматирования используются `UiHelpers`.

Краткая статистика:

1. `MainWindow::RefreshStatisticsPanel` перебирает счета.
2. Считает количество счетов каждого типа.
3. Считает общий положительный баланс.
4. Считает общий долг.
5. Считает количество операций в историях.
6. Записывает результат в `statsLabels`.

Окно статистики:

1. `StatisticsDialog` открывается из `MainWindow`.
2. Кнопка таблицы вызывает `StatisticsVisualizer::getAccountTable`.
3. Кнопка графика балансов вызывает `StatisticsVisualizer::getBalanceChart` и включает `showBalanceGraphic`.
4. Кнопка статистики типов вызывает `StatisticsVisualizer::getAccountTypeChart`.
5. Текстовый результат выводится в `outputEdit`.

GDI-график:

1. Смотреть `StatisticsDialog.cpp`.
2. Обработка рисования находится в `WM_PAINT`.
3. Внутри `WM_PAINT` вызываются `BeginPaint`, `DrawBalanceChart`, `EndPaint`.
4. `DrawBalanceChart` использует `Rectangle`, `TextOutW`, `CreateSolidBrush`, `SelectObject`, `DeleteObject`.
5. Положительные балансы рисуются зеленым, отрицательные - красным.
6. При изменении размера окна `WM_SIZE` пересчитывает `chartRect` и вызывает `InvalidateRect`, чтобы график перерисовался.

## 8. Возможные вопросы на защите

**Зачем нужен класс `Account`?**  
Он задает общий интерфейс для всех счетов: ID, владелец, баланс, история, пополнение, снятие, проценты, сохранение. Благодаря этому `Bank` может работать с разными счетами одинаково.

**Зачем нужен `virtual`?**  
Чтобы при вызове метода через указатель или ссылку на `Account` выполнялась версия метода конкретного наследника.

**Почему `Account` абстрактный?**  
Потому что у него есть чисто виртуальные методы: например, `withdraw`, `applyInterest`, `getType`, `serialize`. Нельзя создать "просто счет", нужен конкретный тип.

**Где используется полиморфизм?**  
В `Bank`: счета хранятся как `std::shared_ptr<Account>`, но при вызове `withdraw`, `applyInterest`, `serialize` выполняется метод `SavingsAccount`, `CheckingAccount` или `CreditAccount`.

**Зачем `std::shared_ptr`?**  
Чтобы хранить объекты наследников через указатель на базовый класс и автоматически управлять временем жизни объектов.

**Почему счета хранятся в `std::map`?**  
ID счета является ключом. `std::map` удобно использовать для поиска, удаления и упорядоченного обхода счетов по ID.

**Как работает перевод?**  
`TransferDialog` вызывает `Bank::transfer`, он передает запрос в `Bank::operator+`. Там проверяются сумма, счета и лимиты, затем баланс отправителя уменьшается, баланс получателя увеличивается, а в историю добавляются `Transfer Out` и `Transfer In`.

**Зачем `operator+`?**  
Это демонстрация перегрузки операторов. Перевод оформлен как добавление к банку объекта `TransferRequest`.

**Как работает сохранение?**  
`FileManager::saveBank` открывает файл и выполняет `file << bank`. Оператор `Bank::operator<<` проходит по счетам и вызывает виртуальный `serialize`.

**Как работает загрузка?**  
`FileManager::loadBank` открывает файл и выполняет `file >> bank`. Оператор `Bank::operator>>` читает тип счета, создает нужный объект и восстанавливает историю транзакций.

**Как работает шаблон `Transaction<T>`?**  
Это класс операции, где тип суммы задается параметром `T`. В проекте используется `Transaction<double>`.

**Как обрабатываются исключения?**  
Банковская логика бросает `InvalidAmountException`, `InvalidAccountException`, `InsufficientFundsException` или `std::runtime_error`. GUI ловит их и показывает через `DialogUtils::ShowException`.

**Что такое овердрафт?**  
Это разрешенный уход баланса в минус. В `CheckingAccount` можно снять больше, чем есть на счете, но не ниже `-overdraftLimit`.

**Как начисляются проценты?**  
У накопительного счета проценты увеличивают положительный баланс. У расчетного счета ничего не происходит. У кредитного счета проценты начисляются на отрицательный баланс и увеличивают долг.

**Как работает кредитный счет?**  
Он разрешает отрицательный баланс до `-creditLimit`. Если баланс отрицательный, `applyInterest` добавляет проценты к долгу.

**Где используется WinAPI?**  
В `main.cpp`, `MainWindow.cpp`, всех диалогах и `DialogUtils.cpp`: создание окон, кнопок, таблицы, обработка сообщений.

**Где используется GDI?**  
В `StatisticsDialog.cpp`, метод `DrawBalanceChart` и обработчик `WM_PAINT`.

**Как поменять кнопку в интерфейсе?**  
Найти `CreateButton` в `MainWindow.cpp` или нужном dialog-файле и изменить текст, ID, координаты или размеры.

**Где создается главное окно?**  
В `MainWindow::Create`, через `CreateWindowExW`.

**Где находится цикл сообщений?**  
В `MainWindow::Run`, где используются `GetMessageW`, `TranslateMessage`, `DispatchMessageW`.

**Где обновляется таблица счетов?**  
В `MainWindow::RefreshAccountTable`.

**Где обновляется краткая статистика главного окна?**  
В `MainWindow::RefreshStatisticsPanel`.

**Где выполняется автозагрузка?**  
В `MainWindow::AutoLoad`, вызывается из `CreateControls`.

**Где выполняется автосохранение?**  
В `MainWindow::AutoSave`, вызывается после успешных изменяющих операций и при закрытии.

**Почему `canWithdraw` protected?**  
Это внутреннее правило счета. Его используют наследники и `Bank` как дружественный класс для перевода, но обычный внешний код не должен напрямую управлять этой проверкой.

**Зачем `friend class Bank` в `Account`?**  
Чтобы `Bank` мог при переводе вызвать защищенные методы `canWithdraw` и `changeBalance`, не создавая обычные операции пополнения/снятия.

**Почему при переводе не вызываются `deposit` и `withdraw`?**  
Чтобы в истории не появились обычные `Deposit` и `Withdraw`. Вместо этого создаются специальные записи `Transfer In` и `Transfer Out`.

**Где создается выписка?**  
В `Bank::generateStatement`, а путь формируется в `StatementDialog.cpp` или `MainWindow::GenerateStatementForAccount`.

**Где открывается папка выписок?**  
В `StatementDialog.cpp` и `MainWindow.cpp`, через `ShellExecuteW(hwnd, L"open", L"statements", ...)`.

**Почему есть `DialogUtils`?**  
Чтобы не повторять во всех диалогах код создания контролов, чтения чисел, конвертации строк и показа ошибок.

**Почему есть `UiHelpers`?**  
Чтобы вынести форматирование отображения счета: тип, деньги, проценты, лимиты, краткое описание.

**Что делает `StatisticsVisualizer`?**  
Готовит текстовую таблицу и текстовые диаграммы. А графический GDI-график находится в `StatisticsDialog.cpp`.
