# Исправления в реальном времени

## 1. Что было исправлено

Были исправлены две небольшие проблемы в текущем проекте BankVS.

1. В разделе "Выбранный счёт" находились лишние кнопки быстрых действий: "Пополнить", "Снять", "История", "Выписка". Эти действия уже есть в левой панели главного окна, поэтому кнопки в карточке выбранного счета дублировали интерфейс.

## 3. Исправление удаления выписки

Удаление выписки добавлено в файл `DeleteAccountDialog.cpp`.

Удаление счета по-прежнему происходит в методе внутреннего класса диалога:

- `Dialog::DeleteAccount`.

Сначала выполняется основное действие:

```cpp
bank.removeAccount(id);
```

Только после успешного удаления счета формируется путь к выписке:

```cpp
std::filesystem::path("statements") /
    ("statement_" + std::to_string(id) + ".txt")
```

Для удаления используется `std::filesystem`, потому что это стандартный способ C++17 работать с путями и файлами:

- `std::filesystem::path` - безопасно собирает путь;
- `std::filesystem::exists` - проверяет наличие файла;
- `std::filesystem::remove` - удаляет файл.

Если файла выписки нет, это не считается ошибкой: программа просто продолжает выполнение.

Если файл есть, но удалить его не удалось, счет обратно не восстанавливается. Пользователь получает предупреждение:

```text
Счёт удалён, но файл выписки удалить не удалось.
```

Пример:

- удаляется счет ID `5`;
- программа удаляет счет через `bank.removeAccount(5)`;
- затем пытается удалить файл `statements/statement_5.txt`;
- если файла нет, удаление счета считается успешным;
- если файл удалить не удалось, счет все равно остается удаленным, но выводится предупреждение.

## 4. Почему это безопасное изменение

Это безопасное изменение, потому что банковская логика `Account` и `Bank` не переписывалась без необходимости.

Удаление выписки выполняется только после успешного вызова `bank.removeAccount(id)`. Если удаление счета не удалось, код удаления файла не выполняется.

Отсутствие файла выписки не вызывает ошибку. Это важно, потому что у счета могла вообще не создаваться выписка.

Если удаление файла выписки завершилось ошибкой, программа не пытается вернуть счет обратно. Это соответствует требованию: счет уже успешно удален, а проблема с файлом считается отдельным предупреждением.

Интерфейс продолжает обновляться через уже существующий `mutatingOutputCallback` в `MainWindow::ProcessCommand`: после удаления обновляются таблица счетов, статистика, выбранный счет, журнал и статус.

Автосохранение продолжает работать как раньше. После успешного удаления диалог вызывает `outputCallback`, а в главном окне для удаления используется callback, который вызывает `AutoSave`.

Генерация выписок не менялась. Папка `statements` и формат имени `statement_ID.txt` остались прежними.

## 5. Как проверить исправления

Проверка удаления кнопок:

1. Запустить приложение.
2. Выбрать счет в таблице.
3. Проверить раздел "Выбранный счёт".
4. Убедиться, что информация о счете отображается.
5. Убедиться, что кнопок "Пополнить", "Снять", "История", "Выписка" в этом разделе больше нет.
6. Убедиться, что кнопки в левой панели "Операции" остались.
7. Выбрать другой счет в таблице и убедиться, что информация в разделе "Выбранный счёт" обновляется.

Проверка удаления выписки:

1. Создать счет.
2. Сгенерировать по нему выписку.
3. Убедиться, что файл `statements/statement_ID.txt` появился.
4. Удалить этот счет через кнопку "Удалить счёт".
5. Проверить папку `statements`.
6. Убедиться, что файл `statement_ID.txt` удален.
7. Проверить, что остальные выписки не удалились.
8. Перезапустить приложение и убедиться, что удаленный счет не восстановился из `bank_data.txt`.
